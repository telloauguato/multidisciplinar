The project is organized into distinct folders for easy navigation and development. In the root directory, you will find the following folders: `app`, `components`, `contents`, `public`, `lib`, and `styles`. Each folder serves a specific purpose, along with important files like `package.json` (for installed dependencies) and `tailwind.config.ts` (for customizing Tailwind CSS).

## Project Structure Overview

### app

This folder contains all Next.js application code, including the main layout and file-based routing for the `docs` and `blog` sections.

```
app                       // Main Next.js app
├── page.tsx             // Hero page
├── layout.tsx           // Main layout
├── blog
│   ├── page.tsx         // Lists all blog posts
│   └── [slug]
│       └── page.tsx     // Specific blog post page
└── docs
    ├── layout.tsx       // Documentation layout with left panel
    └── [[...slug]]      // Catch-all route for documentation pages
```

### components

This folder contains all reusable components, including ShadCN components, custom Markdown components, and general components.

```
components                  // Reusable components
├── ui                     // ShadCN components
├── markdown                // Custom Markdown components
└── navbar.tsx             // General navigation component
```

### styles

This folder contains CSS files, including global styles and additional stylesheets for specific components.

```
styles                     // CSS files
├── globals.css            // Global CSS file
└── syntax.css             // Syntax highlighting styles
```

### contents

This folder is for all Markdown content used for blogs and documentation.


```
contents                   // Markdown content for blogs and documentation
├── docs                   // Documentation content
└── blogs                  // Blog content
```

### public

This folder holds all publicly accessible files, such as images and videos. Ensure that sensitive content is not placed here, as these files can be accessed directly via URLs.

### lib

This directory contains utility or helper functions, including the logic for parsing Markdown content and the routing configuration for documentation.

```
lib                        // Utility or helper functions
├── markdown.ts            // Logic for parsing documentation and blog content
├── routes-config.ts       // Routing configuration for documentation
└── utils.tsx              // Helper functions
```
